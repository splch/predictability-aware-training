## Review — Red-Team Round 2

### Correct (verified, with evidence)
- **Fixed-eval protocol is correctly implemented**: eval batches are drawn once from the deterministic stream before the training loop and are therefore never trained on and identical across runs (`train.py:107`). The Exp 4/6 reported hit@k values match the run logs (`run_tierA2.log:1398,1509`: 0.829 base-posthoc vs 0.890 joint-posthoc).
- **Posthoc re-init and freeze fixes are present** (`train.py:85-90`).
- **The +6pt is not a marginal-popularity artifact** (attack 4, cheap exculpatory check): on the trace data, corpus-level expert usage is near-uniform for both backbones (marginal entropy 2.764 base vs 2.767 joint, max ln16=2.773), and a trivial per-layer "2 most popular experts" predictor hits 0.156 (base) vs 0.148 (joint) — the joint model is if anything marginally *more* uniform, so the hit@k gain is not the router putting more mass on fewer experts.
- **No token-identity degeneracy** (attack 3, CPU probe on `ckpt_A_base.pt` / `ckpt_A_lam0.3.pt`, 6 eval batches): a token-ID→expert lookup predictor hits 0.502 (base) vs 0.511 (joint) — joint routing is no more token-ID-determined than baseline. Persistence predictor ~0.12-0.13 both — no temporal copying.

### Blocker
- **`dump_traces.py:36,41` — trace tensors are scrambled; Experiment 5's trace-driven results are invalid.** `torch.stack(out["topks"], dim=1)` on a list of `[B,T,k]` tensors produces `[B,L,T,k]`, then `.reshape(-1, L, k)` silently interleaves layers and tokens (correct: `stack(dim=2)` or permute before reshape). Proof: per-"layer" marginal usage is identical across all 12 pseudo-layers; adjacent-row top-2 overlap is 0.152 ≈ independent-draw level; hit@k computed from the traces is chance (0.121 h=1) vs 0.829 from the same checkpoint via the correct in-training eval path. Consequences: (a) the toy-geometry table's *learned-prefetch* rows used a chance-level predictor — "learned predictor LOSES to demand" (62-72 vs 85.8 tok/s) reflects the bug, not the measured 0.83/0.89 predictors; (b) even the demand/oracle rows ran on locality-destroyed routing sequences, biasing FIFO-cache behavior; (c) only the synthetic-geometry table (hand-set accuracies, Zipf routing) is unaffected — meaning the paper's **only real-data system measurement is currently wrong**. Fix: one-line stack fix, re-dump both traces (GPU minutes), rerun `cache_sim.py`, correct RESULTS.md Exp 5.

### Ranked findings (attacks 1-6)

1. **HIGH — Headline +6pt is still vs an undermatched control; the isolation test doesn't close this (attacks 1, 6).** The posthoc-on-joint recovery (0.890 vs 0.887) proves predictability is a *backbone property recoverable by this predictor family* — linear heads, soft-CE, 2000 steps, in-distribution stream. It does NOT prove: (a) a stronger posthoc predictor (2-layer MLP + ranking loss per arXiv 2511.10676, trained to convergence) on the *baseline* backbone wouldn't close the +6pt gap — round-1 attack #5 is still open and it threatens the central claim; (b) robustness to int4 quantization, fine-tuning, or domain shift; (c) that an engine training its predictor online/from logs (not 2000 offline steps on the same stream) recovers the gain. **The single most important missing experiment before Tier C is the ranking-aware SOTA control on both backbones.** Also soften "deployable" → "recoverable by a fresh linear predictor in-distribution" in RESULTS.md Exp 4 finding 1.
2. **HIGH — The 3-seed replication varies only weight init (`train.py:30-41`).** `fineweb_stream` has no shuffle buffer and no seed; `torch.manual_seed` doesn't affect stream order. All 3 seeds saw identical data in identical order, and the fixed eval set (16 batches ≈ 65k tokens, stream prefix) is identical across *all* runs in the project. The reported ±0.4pt / ±0.009-nat error bars are init-noise only and understate run variance; eval-set-draw variance is completely unmeasured. Fix: add a shuffle buffer to the stream; draw ≥2 eval sets; report both variance components.
3. **MEDIUM — Quality-cost claims can't transfer from 75x-undertrained models (attack 2).** At 5.77 nats (ppl ~320) the loss is dominated by high-frequency next-token statistics; damage to rare-token/long-tail capability is invisible in an average nat, and the marginal price of a nat rises near convergence — the +0.023±0.009 cost measured here bounds nothing about the tradeoff at a trained loss level. Compounding: the delta is measured on a single fixed 65k-token eval draw. Fix: add a second eval distribution (different FineWeb slice or held-out domain), plot loss curves not just endpoints, run one 4x-token arm to check frontier drift, and scope all quality claims to "at this training horizon."
4. **MEDIUM — Per-token entropy -9%: artifact not fully excluded (attack 4).** Marginal concentration is cleared (above), but sharper per-token routing could still make the linear predictor's job mechanically easier rather than reflecting richer routing structure. Decisive control (one extra arm): train a baseline with an entropy penalty matched to 2.065 nats, then posthoc-predict — if the +6pt survives entropy-matching, it's structure; if it vanishes, it's sharpening. Cheaper proxy: report hit@k stratified by per-token router-entropy buckets.
5. **LOW — Hidden-state homogenization is the degenerate mode your diagnostics miss (attack 3).** My probe: mean off-diagonal cosine of pre-MoE hidden states is 0.051 (base) vs 0.062 (joint, +20% relative). Not collapse, but the predictability loss *does* push hidden states together, and entropy/persist/util_max don't watch this; at λ=1.0 it presumably worsens while all logged diagnostics look "fine." Fix: add a hidden-state effective-rank/cosine diagnostic to `diag`, plus at least one capability probe beyond in-distribution val loss (e.g., loss on a rare-token stratum).
6. **LOW — Data hygiene (attack 5).** `fineweb_stream` concatenates encodings with **no EOS/document delimiter** (`train.py:36-38`): every 512-token window crosses documents, inflating LM loss and injecting artificial routing discontinuities into both the quality metric and the prediction targets (direction of bias unknown). Unshuffled stream ⇒ correlated adjacent chunks and a possibly unrepresentative eval prefix. GPT-2 BPE is a dated tokenizer and differs from OLMoE's vocab — flag as a Tier C transfer caveat. Fix: one-line EOS insertion, shuffle buffer.
