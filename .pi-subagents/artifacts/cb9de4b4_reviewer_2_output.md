## Review

### Q1 — Is Tier C the right next move? Ranked recommendation

**No.** Four cheaper, more decisive items are open (all acknowledged in RESULTS.md "Open items" but now being leapfrogged). Ranked by decisiveness-per-GPU-hour:

1. **Finish the StickyMoE baseline (do first).** RESEARCH.md calls StickyMoE "the most dangerous prior art" and a "**mandatory** training-time baseline," yet the only arm run was a single lambda_sticky=0.1 that "was too weak (entropy rose, nothing moved)" with **no hit@k measured at all** (RESULTS.md Exp 4, finding 5 — "untrained pred"). This is the single largest threat to the re-scoped novelty claim: if a properly tuned sticky-only arm matches joint training on posthoc-recoverable hit@k, the paper's spine collapses. Cost is trivial: Tier A runs at ~35.7k tok/s (run_seeds.log:1506) → 25M tokens ≈ 12 min/arm. A lambda_sticky sweep {0.3, 1, 3} + posthoc-predictor-on-sticky evals + one stacking arm (sticky + lambda_pred=0.3) is under a day. There is no justification for Tier C while this hangs.
2. **Ranking-aware SOTA control (2511.10676 protocol).** The current control is a linear head + soft-CE trained 2000 steps (RESULTS.md red-team #5, "undermatched"). 2511.10676's 2-linear-layer + ranking-aware loss reportedly buys ~15 pts over prior SOTA posthoc. If a matched control recovers even half the +6.2-6.8 pt joint advantage (Exp 6), the effect size narrative changes. Cost: hours — posthoc training on already-frozen checkpoints.
3. **10x longer Tier A run.** The 75-80x-undertrained confound (25M tokens vs ~2B Chinchilla tokens for 99M active) hangs over every claim; undertraining plausibly *inflates* the joint advantage (undertrained routers are high-entropy and malleable; Exp 4 finding 3 already shows the quality-free window narrowing from Tier B to Tier A). At ~12 min per 25M tokens, a 250M-token 3-arm (base/joint/posthoc) run is ~1.5 h/arm plus overhead — the cheapest confound-killer available.
4. **TTFT measurement in cache_sim.** The Exp 5 reframing (finding 5) makes TTFT the candidate headline, but it is explicitly unmeasured. Extending the existing simulator (cold cache, first-token + per-token latency distribution) is hours of work and either substantiates or kills the new framing (see Q4).
5. **Tier C, only after 1-4.** Most expensive, least controlled, and — critically — as currently designed it does not test the core mechanism (see Q2).

### Q2 — Tier C design critique

**The frozen-backbone design undercuts the thesis.** PLAN.md Tier C specifies "router+predictor only, backbone frozen." But the project's differentiating result is Exp 4 finding 1: predictability is a property of the *backbone representations*, recoverable by a fresh posthoc predictor. That mechanism requires predictability-loss gradients flowing into the backbone. With the backbone frozen, gradient reaches only the routers — which is precisely **ReMoE's** design (router-only fine-tune with a gate regularizer, RESEARCH.md training-time table). Tier C as specced risks producing a result that prior art already occupies, and cannot replicate the Exp 4 isolation test in its current form. Concrete fix: unfreeze the last k blocks, or add LoRA adapters on attention/MLP, so representation-shaping is possible; then run the isolation test (fresh posthoc predictor on frozen-fine-tuned vs frozen-stock OLMoE). Even router-only retains *some* value if evaluated that way (routing-level predictability is still deployable), but the PLAN should state which claim each variant proves.

**Target choice is sound.** OLMoE-1B-7B is the right pick: smallest real pretrained MoE, Colibri-supported (RESEARCH.md), 64 experts top-8 drops chance hit@k to 0.125 vs Tier A's 16 experts — a genuinely harder prediction regime, and a *trained* model (3T tokens), which directly probes the undertraining confound. No better target exists on this hardware.

**Feasibility on this box: yes, but PLAN's "hours" is misleading.** Router+predictor-only still requires full fwd+bwd through the 1.3B-active backbone (router grads need input grads): ~6·1.3e9·T FLOPs. At ~15 TFLOPS effective (half the 31 TFLOPS peak from PLAN.md), 25M tokens ≈ 3.6 h — so "hours" holds *only* at a Tier-A-matched token budget, which is thin for moving a pretrained router. 100M tokens ≈ 14 h; 1B tokens ≈ 6 days. **Full fine-tune is memory-infeasible:** 6.9B × 16 B (bf16 weights + fp32 master + AdamW m/v) ≈ 110 GB + gradients/activations > 128 GB; would need CPU optimizer offload and multi-day runs. **Blocker-grade gap: PLAN.md defines no success/failure criteria for Tier C.** Suggested: success = fresh-posthoc hit@8 on fine-tuned > posthoc on stock OLMoE at ≤ small PPL delta and ≥ baseline router entropy; failure = entropy collapse (the Exp 4 finding-4 degeneracy, now measurable) or gain ≤ the router-only/ReMoE-style control.

### Q4 — TTFT reframing: retreat or truth?

**Both, and currently more retreat than truth.** The truth: Exp 5's bandwidth-conservation result is real and rigorously obtained (demand-priority FIFO, prefetch only into idle windows, three modeling bugs fixed en route — verified in cache_sim.py:82-94) — steady-state tok/s gains of +0.5-3% are honest, and the waste-reduction ordering is solid. The retreat: "TTFT may be the headline" is speculation about an unmeasured quantity, published in RESULTS.md in the same breath as the disappointing throughput numbers. There is also a structural reason to demand the measurement before believing it: for the very first token there is no prior context to predict *from* — the TTFT benefit must come from one-layer-ahead prediction *within* the first token's own layer-by-layer compute (a 19 MB fetch ≈ 6 ms vs ~4 ms/layer dense compute at Colibri geometry, so h=1 prediction can hide most of layers 2..75). Plausible, but it needs the cold-cache experiment. **Settling measurement: extend cache_sim.py with an empty-cache start; report time-to-first-token and p50/p99 per-token latency for demand vs base-acc vs joint-acc at Colibri geometry.** Hours of work; do it before the word "headline" appears anywhere else. The current wording ("may be", "unmeasured here") is acceptably hedged — barely.
